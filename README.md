# WNCMS 大逃殺遊戲（Battle Royale）

本專案是一個基於 WNCMS 打造的回合制大逃殺遊戲，強調事件驅動、策略決策與可擴充的遊戲系統設計。

玩家將在同一張地圖中行動、探索、戰鬥與生存，直到最後一人（或隊伍）存活為止。

---

## 遊戲特色

### 回合制行動系統
每位玩家以回合方式進行行動，例如移動、攻擊、使用道具或觸發事件，使遊戲節奏清楚且具策略性。

### 戰鬥與事件紀錄
所有戰鬥、拾取、死亡與特殊事件皆會產生遊戲紀錄，並以可讀性高的方式呈現，方便回顧每一場對戰過程。

### 道具與成長機制
玩家可在地圖中取得各式道具，影響屬性、戰鬥結果或行動選項，並隨著遊戲進行逐步成長。

### 地圖探索與視野限制
地圖支援座標與視野範圍設計，玩家只能得知周圍有限資訊，增加探索與風險判斷的樂趣。

### 高度模組化設計
遊戲邏輯以模組方式設計，可輕鬆擴充新的行為、事件類型、戰鬥規則或遊戲模式。

---

## 遊戲進行流程

1. **建立遊戲**
   建立一場新的大逃殺遊戲，設定地圖、玩家上限與基本規則。

2. **玩家加入**
   玩家加入遊戲並被配置初始位置與基本狀態。

3. **回合開始**
   遊戲依序進入回合流程，每位玩家在自己的回合中選擇行動。

4. **行動與事件觸發**
   玩家行動可能引發戰鬥、拾取道具、遭遇其他玩家或觸發特殊事件。

5. **狀態結算**
   回合結束後，系統結算戰鬥結果、狀態變化與死亡判定。

6. **遊戲持續**
   遊戲回合不斷推進，玩家數量逐漸減少，競爭與壓力逐步升高。

7. **遊戲結束**
   當只剩下一名（或一組）存活玩家時，遊戲正式結束並產生最終結果紀錄。

## 未來規劃 / Roadmap

- **更多地圖與地形類型**
  增加不同風格的地圖設計，影響移動、戰鬥與策略選擇。

- **特殊事件系統**
  引入隨機或條件觸發的全局事件，改變戰局走向。

- **技能與職業機制**
  為玩家加入職業或技能樹，提供更豐富的玩法差異。

- **AI 與非玩家角色**
  增加怪物或中立角色，提升遊戲世界的動態性。

- **多種遊戲模式**
  支援單人、團隊、生存挑戰或限時模式等不同玩法。

- **視覺化戰鬥與紀錄呈現**
  強化遊戲紀錄與戰鬥過程的視覺呈現，提升觀戰與回顧體驗。

---

## 專案目標

本專案旨在建立一套可重複使用、可持續擴展的大逃殺遊戲基礎，適合作為策略遊戲、文字冒險或模擬類遊戲的核心架構。
